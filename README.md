# HTCBot

Telegram-бот для Церкви Святой Троицы. Предоставляет информацию о богослужениях, малых группах, календаре событий и другие полезные функции.

## Архитектура

Проект организован по принципу **Feature-First**: каждая функциональность — самостоятельный модуль со своими текстами, клавиатурами, константами и утилитами.

### Структура проекта

```
src/
├── app.ts                    # Точка входа: создание бота, регистрация фич
├── config/                   # Конфигурация (env)
├── constants/                # Глобальные константы (кнопки меню, навигация)
├── services/                 # Бизнес-логика (календарь, Buildin API, доступ)
├── utils/                    # Утилиты (текст, логирование, загрузка, guards)
├── types/                    # TypeScript типы
├── middlewares/              # Middleware (авторизация, обработка ошибок)
└── features/                 # Фичи (основные модули функциональности)
    ├── start/
    ├── main-menu/
    ├── church-calendar/
    ├── small-groups/
    └── ...
```

### Структура фичи

Каждая фича в `features/{feature-name}/` может содержать:

- **`{feature-name}.feature.ts`** — логика (хендлеры, рендеринг, регистрация)
- **`{feature-name}.texts.ts`** — тексты (организованы на 3 секции: block тексты, динамические функции, основной объект)
- **`{feature-name}.keyboard.ts`** — клавиатуры (опционально)
- **`{feature-name}.constants.ts`** — константы кнопок (опционально)
- **`{feature-name}.util.ts`** — утилиты фичи (опционально)

### Правила организации

#### Глобальное vs Локальное

- **Глобальное** (используется в 2+ фичах) → `services/`, `utils/`, `constants/`
- **Локальное** (используется в 1 фиче) → в папке фичи

**Правило "2+ использования"**: если функция/текст/константа используется в нескольких фичах — выносим в глобальное.

### Как добавить новую фичу

1. Создайте папку `src/features/my-feature/`
2. Создайте файлы (минимум `my-feature.feature.ts` с функцией `registerMyFeature`)
3. Зарегистрируйте в `app.ts`: `import { registerMyFeature } from "./features/my-feature/my-feature.feature"; registerMyFeature(bot);`

### Форматирование текстов

Проект использует `@grammyjs/parse-mode` с функцией `fmt` для работы с MarkdownV2. Тексты организованы в три секции:

- **Block тексты** — большие многострочные константы
- **Динамические тексты** — функции, генерирующие тексты
- **Основной объект** — inline тексты и ссылки на block/динамические

### Разработка

```bash
npm install
npm run dev      # Разработка
npm run build    # Сборка
```

Все переменные окружения настраиваются через `.env` файл (см. `src/config/env.ts`).

### Обновление переменных окружения

#### Локально (перед деплоем)

1. **Обнови JSON файлы в `src/secrets/`:**
   - `src/secrets/leaders.json` — данные лидеров
   - `src/secrets/small-group-addresses.json` — адреса малых групп

2. **Закодируй JSON в base64:**
   ```bash
   npx ts-node src/scripts/encode-env.ts
   ```
   
   Скрипт выведет готовые строки:
   ```
   LEADERS_JSON_BASE64=eyJpZDEiOns...
   GROUP_ADDRESSES_JSON_BASE64=eyJncm91cE5hbWUiOls...
   ```

3. **Обнови `.env` локально:**
   - Скопируй вывод `LEADERS_JSON_BASE64` и `GROUP_ADDRESSES_JSON_BASE64` в `.env`
   - Для `AUTHORIZED_USERNAMES` просто укажи строку через запятую: `username1,username2,@username3`

#### На сервере

После того как обновил `.env` локально и закоммитил изменения:

1. Подключись к серверу
2. Обнови `.env` файл на сервере:

   ```bash
   # Переходим в директорию проекта
   cd htc-bot/
   
   # Открываем .env файл в редакторе nano
   nano .env
   ```
   
   **Как работать в nano:**
   - Перемещение: стрелки на клавиатуре
   - Перейти в начало строки: `Ctrl + A`
   - Перейти в конец строки: `Ctrl + E`
   - Редактирование: просто печатай
   - Удалить строку целиком: `Ctrl + K` (вырезать строку), затем `Ctrl + U` (вставить обратно, если ошибся) или просто продолжить — строка удалена
   - Заменить текст: `Ctrl + \` (поиск и замена), введи что искать, затем на что заменить, затем `Y` для каждого вхождения или `A` для всех
   - Поиск: `Ctrl + W`
   - Сохранить и выйти: `Ctrl + O` (сохранить), затем `Enter` (подтвердить), затем `Ctrl + X` (выйти)
   - Выйти без сохранения: `Ctrl + X`, затем `N` (нет)
   
   Найди нужные строки (`LEADERS_JSON_BASE64`, `GROUP_ADDRESSES_JSON_BASE64`, `AUTHORIZED_USERNAMES`) и замени их значениями из локального `.env`.

3. Перезапусти бота (см. раздел "Обновление на сервере")

**Важно:** Файлы в `src/secrets/` не попадают в репозиторий (они в `.gitignore`), поэтому на сервере используется только `.env` с закодированными значениями.

### Обновление на сервере

1. Подключитесь к серверу (Timeweb Cloud или другой)
2. Обновите `.env` файл на сервере (если изменяли переменные окружения):
   - Отредактируйте `.env` вручную через `nano .env` или `vim .env`
   - Или скопируйте содержимое из локального `.env` файла
3. Выполните следующие команды:

```bash
# Переходим в директорию проекта
cd opt/htc-bot/

# Проверяем текущий статус репозитория (изменения, несохраненные файлы)
git status

# Получаем информацию об изменениях с удаленного репозитория
git fetch origin

# Загружаем изменения из ветки master
git pull origin master

# Собираем TypeScript в JavaScript
npm run build

# Запускаем/перезапускаем бота через PM2
pm2 start dist/app.js --name htc-bot --env production
pm2 restart htc-bot

# Просмотр логов бота (для проверки работы)
pm2 logs htc-bot
```
